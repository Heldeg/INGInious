Understand your result.

Here you will understand what means the result you receive from your submission.

After you submit the code, you will get an overall result, the grade and the result corresponding to each test case.
Results related to your code. These are the different results you can get after assessing your code

  • ACCEPTED

    You can breath with this result. This means that your output is the same as the expected output.

  • WRONG ANSWER

    This result means that the output given by your code does not coincide with the expected output.
    In some cases you can see the difference of your output and expected output. So that way, you can determine what is going on in your code.
    TIPS: Check again your code. Try using the custom input (tools button).

  • COMPILATION ERROR

    This message means that your code does not compile, that means there is something wrong with your code. In order to help you with that, there you can see some help telling you what is wrong with your code and where you can find the error.

  • RUNTIME ERROR

    Runtime error means that your code failed while it was running, so your code exits unexpectedly. There are many reasons that may cause your code to crash. Some of them are: division by zero, segmentation fault, invalid memory access, null pointers, among others.

  • TIME LIMIT

    When your submission times out, means that your code took too long to run the input test cases. There are many reasons may cause this, some of them are: infinite loops, you are considering a not optimal solution for the problem, among others.
    TIPS: Please check your code and debug it, you might be missing something. If it is not an optimal solution try to change the idea.

  • MEMORY LIMIT EXCEEDED

    Getting this result means that your code is using too much memory while it is running. There are tons of causes, a possible one might be that you are storing too much data more than the possible storable, among others.
    TIPS: Check the data structures you are using or think a more optimal solution that uses less memory.
Results related to UNCode. These are some unexpected errors that came up while trying to assess your submission.

  • INTERNAL ERROR

    If you get this message, do not worry, THERE'S NOTHING WRONG WITH YOUR CODE.
    This error means that UNCode had a problem while running or assessing your code. So the problem is on our side.
    Please reload the page and try submitting again! If you still get the same result, please wait a while and try again.

  • SUBMISSION TIMED OUT

    If you get this, means that we had a problem running and assessing your code. We had to kill the process.
